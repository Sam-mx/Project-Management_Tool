Project Architecture: Samwise PM Platform1. Executive SummarySamwise is an AI-enhanced, open-source project management tool designed to bridge the gap between simple, visual Kanban tools (like Trello) and complex, enterprise-heavy systems (like Jira). By integrating Generative AI (Google Gemini) directly into a MERN-stack environment, Samwise automates routine tasks and provides proactive decision-making support, reducing cognitive load for small teams and individual developers. 2. System Architecture FlowThe system utilizes a client-server architecture where the frontend (React) communicates via REST APIs for persistent storage and Socket.io for live event broadcasting. 4. Database Schema (ERD)The logical data model follows a strict hierarchy (User → Space → Board → List → Card). I utilized a hybrid approach to normalization: Normalization (References): Users and Members are referenced by ObjectId to ensure data consistency across the platform. Denormalization (Embedding): Comments and ChecklistItems are embedded within the Card document to optimize read performance and reduce expensive join operations. 5. Agile/Management WorkflowThe project adhered to the Project Management Life Cycle (PMLC) to ensure academic rigor and strict deadline adherence: Planning: Utilized a Work Breakdown Structure (WBS) to decompose the system into distinct modules: System Core, Frontend, Real-time Service, AI Services, and Documentation. Agile Integration: While the overarching framework was PMLC, the development philosophy prioritized Agile principles—iterative modularity, rapid prototyping, and continuous feedback. Quality Assurance: Implemented a multi-tier testing strategy, including Unit Testing (logic validation), Integration Testing (API contract validation), System Testing (real-time synchronization), and User Acceptance Testing (UAT).
